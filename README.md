# 🕸️ wayback.sh

A passive recon automation script that collects historical and live URLs for a target domain, discovers endpoints and parameters, probes for live hosts, and categorizes findings using pattern matching — all in a single command.

```bash
./wayback.sh domain.com
```

---

## What It Does

Runs a full passive URL collection and analysis pipeline:

1. **gau** — pulls URLs from AlienVault OTX, Wayback Machine, Common Crawl, and URLScan
2. **waybackurls** — fetches archived URLs from the Wayback Machine
3. **waymore** — extended wayback collection across multiple sources
4. **xnLinkFinder** — extracts endpoints, parameters, and a target-specific wordlist from collected URLs and waymore archived responses
5. **httpx** — probes the final URL list for live hosts, capturing status codes, page titles, and tech stack
6. **gf** — runs pattern matching across live URLs to surface interesting findings (SQLi, XSS, SSRF, LFI, open redirects, and more)

## Dependencies

The script will **automatically check for and install** missing tools on first run.

| Tool | Type | Source |
|------|------|--------|
| [gau](https://github.com/lc/gau) | Go | `go install github.com/lc/gau/v2/cmd/gau@latest` |
| [waybackurls](https://github.com/tomnomnom/waybackurls) | Go | `go install github.com/tomnomnom/waybackurls@latest` |
| [gf](https://github.com/tomnomnom/gf) | Go | `go install github.com/tomnomnom/gf@latest` |
| [httpx](https://github.com/projectdiscovery/httpx) | Go | `go install github.com/projectdiscovery/httpx/cmd/httpx@latest` |
| [waymore](https://github.com/xnl-h4ck3r/waymore) | Python | `pip install waymore` |
| [xnLinkFinder](https://github.com/xnl-h4ck3r/xnLinkFinder) | Python | `pip install xnLinkFinder` |

> **Requirements:** Go and Python/pip must be installed before running. Git is required for gf pattern installation.

### gf Patterns

On first run, the script automatically installs two pattern sets into `~/.gf/`:

- [tomnomnom/gf](https://github.com/tomnomnom/gf) — default patterns
- [1ndianl33t/Gf-Patterns](https://github.com/1ndianl33t/Gf-Patterns) — extended bug bounty pattern set

Any patterns you manually add to `~/.gf/` will also be picked up automatically.

## Output Files

All output files are namespaced to your target domain, so running against multiple targets won't overwrite results.

| File | Description |
|------|-------------|
| `domain.com_gau_wayback_output.txt` | URLs from gau |
| `domain.com_waybackurls_wayback_output.txt` | URLs from waybackurls |
| `domain.com_waymore_wayback_output.txt` | URLs from waymore |
| `domain.com_all_wayback_output.txt` | Merged + deduplicated wayback URLs |
| `domain.com_endpoints.txt` | Endpoints discovered by xnLinkFinder |
| `domain.com_parameters.txt` | Potential parameters discovered by xnLinkFinder |
| `domain.com_wordlist.txt` | Target-specific wordlist generated by xnLinkFinder |
| `domain.com_final.txt` | Final deduplicated URL list (wayback + endpoints) |
| `domain.com_wayback_httpx_check.txt` | Live URLs with status codes, titles, and tech detection |
| `domain.com_gf_results/` | Directory of gf pattern match files, one per pattern |

---

## gf Pattern Categories

Pattern matching runs against live URLs and outputs a separate file per matched pattern. Categories include (but are not limited to):

- `sqli` — SQL injection parameters
- `xss` — Cross-site scripting vectors
- `ssrf` — Server-side request forgery
- `lfi` — Local file inclusion
- `rce` — Remote code execution
- `idor` — Insecure direct object references
- `redirect` — Open redirects
- `ssti` — Server-side template injection
- `cors` — CORS misconfigurations
- `s3-buckets` — S3 bucket references
- `interestingparams` — Interesting parameter names
- `interestingEXT` — Interesting file extensions
- + all patterns from the 1ndianl33t extended set

---

## Notes

- `waymore` saves its results to `~/.config/waymore/results/<domain>/` — the script copies these into the working directory automatically
- `xnLinkFinder` appends to output files across both runs (URL list pass and waymore directory pass), so parameters and wordlist entries are consolidated
- `httpx` output is used as the input for `gf` — if httpx returns no results, gf falls back to the full final URL list
- Running this script against targets you do not have permission to test may be illegal. Always obtain proper authorization before running recon tools
